# تقرير تحليل نظام Inbox Money Vault (IMV)

هذا التقرير يوضح هيكلية النظام، وظائف الصفحات، وتدفق تجربة المستخدم (User Flow).

---

## 1. نظرة عامة على النظام (System Overview)
نظام **Inbox Money Vault** هو منصة ذكية مخصصة لعمليات المبيعات المباشرة (Direct Sales) ووجلب العملاء المحتملين (Lead Generation). يهدف النظام إلى تسهيل عملية الوصول إلى الشركات المستهدفة عبر البريد الإلكتروني (Cold Outreach) باستخدام تقنيات الذكاء الاصطناعي لتخصيص الرسائل.

---

## 2. تحليل الصفحات (Page-by-Page Analysis)

### أ- صفحات الوصول (Authentication)
*   **تسجيل الدخول والاشتراك (`/login` & `/signup`)**:
    *   **الوظيفة**: البوابة الرئيسية لتأمين النظام.
    *   **التقنية**: تعتمد على Supabase Auth لإدارة الجلسات بشكل آمن.

### ب- لوحة التحكم والعمليات (Dashboard & Core Logic)
*   **لوحة التحكم الرئيسية (`/dashboard`)**:
    *   **الغرض**: عرض ملخص سريع للبيانات (عدد العملاء، الرسائل المنشأة، التنبيهات).
    *   **التدفق**: تعتبر نقطة الانطلاق لأي عملية داخل السيستم.

*   **توليد العملاء (`/leads`)**:
    *   **الغرض**: المحرك الرئيسي للنظام لجلب بيانات الشركات.
    *   **الوظيفة**: يدخل المستخدم "المجال" و "الموقع"، ويقوم النظام بتخصيص قائمة من العملاء تتضمن (اسم الشركة، البريد الإلكتروني، الرابط الإلكتروني).
    *   **القيود**: يوجد نظام "حدود يومية" (Daily Limits) لضمان جودة البيانات وعدم إساءة الاستخدام.

*   **منشئ الرسائل الذكي (`/email-builder`)**:
    *   **الغرض**: تحويل بيانات العميل إلى رسالة إقناع احترافية.
    *   **الوظيفة**: يأخذ معلومات العميل من صفحة الـ Leads ويستخدم الذكاء الاصطناعي لكتابة "مسودة" بريد إلكتروني مخصصة تزيد من نسب الرد.

*   **تعليمات الإرسال (`/send-instructions`)**:
    *   **الغرض**: ضمان نجاح وصول الرسائل.
    *   **المحتوى**: دروس وإرشادات حول كيفية إعداد البريد الإلكتروني المهني وتجنب مرشحات السبام (Spam Filters).

*   **سجل النشاط (`/activity`)**:
    *   **الغرض**: مراقبة الأداء التاريخي.
    *   **الوظيفة**: يعرض جدولاً زمنياً لكل العمليات التي قام بها المستخدم (جلب عملاء، إنشاء رسائل).

*   **العروض والترقية (`/offers`)**:
    *   **الغرض**: تطوير حساب المستخدم.
    *   **الوظيفة**: عرض الباقات المتاحة لزيادة حدود البحث اليومية (Daily Allocation Limits).

---

## 3. تدفق العمل المثالي (The Ideal User Flow)

1.  **المرحلة الأولى (Acquisition)**: يذهب المستخدم لصفحة `Leads` ويبحث عن قطاع معين (مثلاً: مطاعم في دبي).
2.  **المرحلة الثانية (Selection)**: يختار المستخدم شركة من النتائج ويضغط "Generate Email".
3.  **المرحلة الثالثة (Creation)**: يتم تحويله لـ `Email Builder` حيث يحصل على رسالة جاهزة ومهنية.
4.  **المرحلة الرابعة (Execution)**: يراجع `Send Instructions` لضمان الإرسال الصحيح، ثم يقوم بالتواصل مع العميل.
5.  **المرحلة الخامسة (Monitoring)**: يراقب تقدمه من خلال الـ `Dashboard` و `Activity`.

---

## 4. المكونات التقنية (Technical Infrastructure)
*   **Frontend**: Next.js (App Router) مع Tailwind CSS للتصميم.
*   **Backend**: Supabase (Database, Auth, Edge Functions).
*   **Animations**: Framer Motion لتجربة مستخدم سلسة وراقية.
*   **Icons**: Lucide React.
